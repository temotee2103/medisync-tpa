# Claim Lifecycle

## Status definitions (unified)
- draft: created but not submitted
- submitted: submitted by member/provider, awaiting admin review
- in_process: reviewed and queued for payout processing
- approved: payout completed and proof recorded
- rejected: rejected with required notes
- more_information: requester must provide additional info

## Approval gates
- Member reimbursements require a member payout profile before completion.
- Provider claims require provider payout profile and payment proof before completion.

## Roles
- Admin reviews and can move submitted → in_process / rejected / more_information
- Accountant completes in_process → approved after payout proof upload
